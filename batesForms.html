WHY LABELS THAT WAY?

When I needed to float labels inline, and have helper tooltips and validation tooltips, there were issues with positioning the label. The labels can go after the inputs and be accessible so long as each label uses the "for" attribute

Also, with this layout the labels don't need the occasional z-index that they used to need for some inputs--like the mutli-select box.

In the current Papermate, tabs are used to house a group of inputs that are much more related to one another than they are to the main form. For instance, on a Create Story page, there is an "OPTIONS" tab that shows fields for Special Advertising.

There are a couple reasons Tabs are discouraged

  • Areas that might be important to some users might be hidden by default
  • It becomes unclear to the user what is being saved--whether the save/cancel button saves everything across all tabs, or if it is necessary to save a given tab's content individually
